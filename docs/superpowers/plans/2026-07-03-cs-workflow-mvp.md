# CS 업무 흐름 MVP 구현 계획

> **작업 원칙:** 이 계획은 `main` 브랜치에서 바로 실행한다. 자동 처리 결과는 확정값이 아니라 CS 담당자가 확인할 초안으로 다룬다.

**목표:** 현재 Streamlit 기반 CS 카테고리 분류 MVP를 `전사문 입력/추출/분류/리뷰/저장/메일 초안`까지 이어지는 시험판 업무 흐름으로 확장한다.

**구조:** 지금은 Streamlit 앱을 유지하되, 추출/저장/메일 초안/STT 관련 로직은 `src/` 모듈로 분리한다. 이렇게 해두면 나중에 공식 사내 사이트로 전환할 때 화면만 교체하고 핵심 업무 로직은 재사용하기 쉽다.

**기술:** Python, Streamlit, pandas, SQLite, pytest, 기존 키워드 기반 분류기, 무료 로컬 STT 후보.

---

## 현재 `cs-` 폴더의 의미

`cs-`는 GitHub 저장소 `mincyann/cs-`를 이 컴퓨터로 받아온 로컬 프로젝트 폴더다. 부모 폴더에 있는 엑셀/PDF 업무 자료와 달리, 이 폴더는 CS 자동 분류 웹앱 코드와 테스트를 담고 있다.

주요 파일은 다음과 같다.

- `app.py`: Streamlit 웹앱 화면과 현재 업무 흐름
- `src/classify.py`: 이슈/대응 텍스트 카테고리 자동 분류
- `data/categories.json`: 카테고리와 키워드 정의
- `data/sample_cases.json`: 안전한 샘플 상담 데이터
- `src/result_store.py`: 기존 CSV 저장 로직
- `tests/`: 자동 테스트

현재 작업 중인 변경사항에는 OpenAI STT 초안이 포함되어 있다. 앞으로는 유료 OpenAI STT를 기본값으로 쓰지 않고, 무료 로컬 STT와 샘플 전사문 중심 흐름으로 바꾼다.

## Task 1. 전사문에서 이슈/대응 추출

**목표:** 전사문에서 고객 문의와 상담사 대응을 분리해 이슈/대응 초안을 만든다.

**파일**

- 생성: `src/transcript_extract.py`
- 생성: `tests/test_transcript_extract.py`

**구현 내용**

- `extract_issue_response(transcript_text)` 함수 추가
- `고객:`, `수검자:`, `병원:` 같은 라벨은 이슈로 분류
- `상담사:`, `CS:`, `담당자:` 같은 라벨은 대응으로 분류
- 라벨이 없으면 전체 전사문을 이슈 초안으로 넣고 리뷰 필요 표시

**검증**

- 라벨 있는 전사문에서 이슈/대응이 분리되는지 확인
- 짧고 불명확한 전사문은 리뷰 필요로 표시되는지 확인

---

## Task 2. 책임 주체별 메일 초안 생성

**목표:** 리뷰 완료 전후에 책임 주체에게 보낼 메일 초안을 만든다.

**파일**

- 생성: `src/email_draft.py`
- 생성: `tests/test_email_draft.py`

**구현 내용**

- `build_email_draft(...)` 함수 추가
- 제목 형식: `[CS 이슈 공유] {카테고리} - {짧은 이슈 요약}`
- 본문에는 카테고리, 책임 주체, 이슈 요약, CS 대응, 확인 요청 문구 포함
- 담당 메일이 비어 있으면 수신자 확인 필요로 표시

**검증**

- 메일 제목과 본문에 핵심 정보가 들어가는지 확인
- 수신자 메일이 없을 때 리뷰 필요로 표시되는지 확인

## Task 3. SQLite 리뷰 저장소 추가

**목표:** CS 담당자가 리뷰 완료한 상담 기록을 앱 안에 저장한다.

**파일**

- 생성: `src/case_store.py`
- 생성: `tests/test_case_store.py`

**구현 내용**

- `data/reviewed_cases.db`에 SQLite 저장
- 저장 필드: 전사문, 이슈, 대응, 추천 카테고리, 최종 카테고리, 책임 주체, 리뷰 상태, 리뷰어, 내부 메모
- 저장된 리뷰 목록을 최신순으로 조회

**검증**

- 리뷰 완료 케이스를 저장하고 다시 조회할 수 있는지 확인

## Task 4. Streamlit 화면에 전체 흐름 연결

**목표:** 앱에서 전사문 입력부터 리뷰 저장과 메일 초안 생성까지 한 화면 흐름으로 이어지게 한다.

**파일**

- 수정: `app.py`

**구현 내용**

- 전사문 입력 영역 유지
- `전사문에서 이슈/대응 추출` 버튼 추가
- 추출 결과를 기존 `이슈`, `대응` 입력칸에 자동 반영
- 기존 카테고리 분류 버튼으로 자동 분류
- 최종 카테고리, 책임 주체, 리뷰어, 내부 메모 입력
- `리뷰 완료 저장` 버튼 추가
- `메일 초안 생성` 버튼 추가
- 저장된 리뷰 목록 표시

**검증**

- 샘플 전사문 붙여넣기
- 이슈/대응 추출
- 카테고리 분류
- 리뷰 저장
- 메일 초안 생성

# CS 이슈/대응 기반 카테고리 자동 분류 웹 MVP

CS 전화문의 또는 상담 기록에서 추출된 이슈와 대응 텍스트를 입력하면, 규칙 기반 Agent가 가장 적절한 카테고리를 추천하는 Streamlit MVP입니다.

## 실행 방법

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 디자인 방향

현재 UI는 Zendesk류 고객지원 콘솔을 벤치마킹한 밝고 신뢰감 있는 화면을 목표로 합니다.

- 상담 입력과 추천 결과를 좌우로 분리
- 추천 카테고리, 신뢰도, 검토 여부를 결과 카드에 강조
- 최근 분류 결과를 하단 표로 정리
- STT 업로드 영역은 향후 기능으로 표시

## 데이터 원칙

- 실제 CS 원본 엑셀, 고객명, 병원명, 담당자명, 전화번호, 음성 파일은 사용하지 않습니다.
- 이 프로젝트는 데모와 개념 검증용입니다.
- 샘플 버튼은 우선 `../outputs/CS_2026_synthetic.xlsx`의 가상 데이터를 읽습니다.
- 가상 엑셀이 없으면 `data/sample_cases.json`의 안전한 샘플을 사용합니다.
- `.gitignore`는 엑셀, 오디오, 원본 데이터 폴더, `.env`를 제외하도록 설정되어 있습니다.

## 현재 MVP 범위

- 이슈/대응 직접 입력
- 가상 엑셀 기반 샘플 입력
- `categories.json` 기반 keyword scoring 분류
- 추천 카테고리, 신뢰도, 사유, 검토 필요 여부, 매칭 키워드 표시
- 사람이 최종 카테고리를 수정할 수 있는 select box
- 최근 분류 결과 표와 CSV 다운로드

## 향후 STT 확장

녹음본 업로드와 자동 STT는 현재 구현하지 않습니다. 향후에는 다음 흐름으로 붙일 수 있습니다.

1. 녹음본 업로드
2. STT로 전체 통화 텍스트 변환
3. AI가 이슈와 대응 초안 작성
4. 상담사가 초안을 수정
5. 기존 카테고리 분류 실행

## 분류 방식

`src/classify.py`의 `classify_issue_response()`가 이슈와 대응 텍스트를 분석합니다.

- 이슈 텍스트 키워드 가중치: 2.0
- 대응 텍스트 키워드 가중치: 1.0
- 점수가 낮으면 `[검토 필요]`
- 1위와 2위 점수가 비슷하거나 검토 신호가 있으면 사람 검토 권장

LLM 분류기로 확장할 때도 같은 함수 인터페이스를 유지하면 앱 변경을 줄일 수 있습니다.
